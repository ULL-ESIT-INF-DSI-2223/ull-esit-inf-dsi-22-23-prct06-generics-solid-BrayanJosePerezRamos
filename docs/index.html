Practica 5: Objetos, Clases e Interfaces

Esta práctica consiste en aprender a utilizar los conceptos de objetos, clases e interfaces en TypeScript, para asi controlar lo básico de la programación orientada a objetos en este lenguaje, para ello se implementaron 2 ejercicios, mas otros dos ejercicios relaizados en la hora de prácticas de la asignatura.

Ejercicios

Ejercicio 1: Biblioteca Musical

Se pide diseñar e implementar el conjunto de clases e interfaces necesarias para almacenar una biblioteca musical, la cual debe cumplir con lo siguiente:

Se debe almacenar la informaciónde cada artista, esta consta de lo siguiente:

  • Nombre
  • Numero de oyentes mensuales
  • Discografia

A su vez, la discografía de cada artista consta de una lista de albums, los cuales tienen lo siguiente:

  • Nombre
  • Año de publicación
  • Lista de canciones

Asi pues las canciones constan de:

  • Nombre
  • Duración en segundos
  • Generos
  • Single (si la cancion fue lanzada como single o no)
  • numero de reproducciones

Teniendo en cuenta todo esto, tendremos que crear la biblioteca musical, la cual tendrá las siguientes funciones:

  • Almacenar la información de diferentes artistas, su discografía y las canciones pertenecientes a cada disco o álbum.
  • Mostrar por la consola la información de la biblioteca en formato tabla (console.table).
  • Permitir llevar a cabo búsquedas de artistas, discos y canciones y mostrar los resultados de la búsqueda en formato de tabla.
  • Permitir calcular el número de canciones incluidas en un disco concreto.
  • Permitir calcular la duración de un disco, a partir de la duración de todas y cada una de las canciones que lo conforman.
  • Permitir calcular el número de reproducciones de un disco, a partir del número de reproducciones de todas y cada una de las canciones incluidas en el mismo.

Con todo lo anterior, mi implementación de la biblioteca musical consta de las clase Cancion, Album, Artista y BibliotecaMusical, para asi dividir la información de cada tipo en sus respectivas clases, y así tener un código más ordenado y fácil de entender.

Ejercicio 2: Conecta 4

Consiste en el clásico juego de mesa Conecta 4, en el que dos jugadores se enfrentan en una partida a ver quien consigue primero alinear 4 fichas de su color en horizontal, vertical o diagonal. El tablero consta de 6 filas y 7 columnas. El juego se desarrolla en turnos, en los que cada jugador coloca una ficha en una columna del tablero, la cual cae hasta la primera fila libre de dicha columna. El objetivo del juego es conseguir alinear 4 fichas de tu color en horizontal, vertical o diagonal. En caso de que el tablero se llene y no se haya conseguido alinear 4 fichas de un mismo color, se declara empate.

Se pide crear una clase Conecta4 que implemente el juego de Conecta 4. La clase debe tener la siguiente funcionalidad:

  • El juego comienza con el Jugador 1 colocando la primera ficha y, en turnos sucesivos, debe ir alternándose con el Jugador 2. Se deberá mostrar por consola a qué jugador le toca colocar una ficha.
  • Si un jugador intenta colocar una ficha en una columna completa, se mostrará un mensaje informando de que la columna está completa y se le permitirá seleccionar otra columna para colocar la ficha. Lo anterior debe repetirse hasta que el jugador coloque su ficha.
  • Una vez que el jugador correspondiente haya colocado una ficha, debe mostrarse por la consola el estado del tablero.
  • Cuando alguno de los dos jugadores gane, se debe informar de lo anterior en la consola y terminar el juego.

Se pide implementar una jerarquía de clases que permita representar el juego de Conecta 4 y que cuente con las funcionalidades previamente descritas. Así pues yo implementé con una interfaz IFicha y una clase Ficha que implementa la interfaz, una clase Rejilla, una clase Jugador y una clase Conecta4 que implementa el juego de Conecta 4.

Tablas de multiplicar

Se pide una función que dado un número N cree un array de arrays con las primeras N tablas de multiplicar cada una con sus primeros N productos (por ejemplo, si N es 3, el array contendrá las tablas de multiplicar del 1 al 3, es decir, 1x1, 1x2, 1x3, 2x1, 2x2, 2x3, 3x1, 3x2, 3x3).

La función debe devolver un array de arrays de números, o undefined si el número N es menor o igual a 0 o no es entero. Mi implementación de la función es la siguiente:

function productTable(numN: number): number[][] | undefined {
  // si el numero es menor o igual a 0 o no es entero, retorna undefined
  if (numN <= 0 || numN % 1 !== 0) {
    return undefined;
  }
  let productTables: number[][] = [];
  // bucle para rellenar el array de arrays
  for (let i = 1; i <= numN; i++) {
    let table: number[] = [];
    // bucle para rellenar cada arry 
    for (let j = 1; j <= numN; j++) {
      table.push(i * j);
    }
    productTables.push(table);
  }
  return productTables;
}

Lo que hace la función es comprobar si el número es menor o igual a 0 o no es entero, si es menor o igual a 0 o no es entero devuelve undefined, si no es menor o igual a 0 o no es entero crea un array de arrays vacío, y crea un bucle que se ejecuta tantas veces como el número que se le pasa por parámetro, en cada iteración del bucle crea un array vacío, y crea otro bucle que se ejecuta tantas veces como el número que se le pasa por parámetro, en cada iteración del bucle añade al array creado en la iteración anterior el producto de la iteración del bucle exterior por la iteración del bucle interior, y al final añade el array creado en la iteración anterior al array de arrays creado en la primera iteración del bucle exterior, y al final devuelve el array de arrays.
